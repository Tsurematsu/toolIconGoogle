This guide covers the optimal, end-to-end process for using the Material Symbols CLI tool to automate your icon workflow.

download

1. Installation

Install the CLI tool globally using your preferred package manager like npm or yarn.

npm install -g material-symbols-cli
tune

2. Configuration

Create a configuration file named `symbols.json` in your project's root directory to define your settings.

touch symbols.json
check_box

3. Icon Selection

Define the list of required Material Symbols in the `symbols.json` file. You can specify names, weights, and styles.

{
  "icons": ["home", "settings", "favorite"],
  "outputDir": "./src/assets/icons",
  "format": "svg"
}
terminal

4. Generation

Run the core command to process the configuration and generate the output files into your specified directory.

material-symbols-cli generate
integration_instructions

5. Integration

Import and use the generated assets in your project. Here's an example for an SVG icon in an HTML file.

<img src="./src/assets/icons/home.svg" alt="Home icon">

You've completed the recommended workflow! For more advanced options and configurations, please refer to our full documentation.
